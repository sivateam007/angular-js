Angular Get Started

Angular ஐப் பயன்படுத்தத் தொடங்குங்கள்

Angular ஐத் தொடங்குதல்

Angular ஐப் பயன்படுத்த, உங்களுக்கு Node.js உடன் சேர்க்கப்பட்டுள்ள npm தேவைப்படும்.

குறிப்பு:

Angular இன் கருவிகள் (CLI மற்றும் கட்டமைப்பு கருவிகள்) Node.js இல் இயங்குகின்றன மற்றும் npm ஐப் பயன்படுத்தி பேக்கேஜ்களை நிறுவுகின்றன.

Node.js ஐச் சரிபார்க்கவும்

Node.js மற்றும் npm நிறுவப்பட்டு உங்கள் PATH இல் உள்ளதை உறுதிப்படுத்த இதை இயக்கவும்:

Node.js பதிப்பைச் சரிபார்க்கவும்

node -v

இது உங்கள் நிறுவப்பட்ட Node.js பதிப்பை அச்சிடும், எடுத்துக்காட்டாக:

v20.x.x

பதிப்பு பரிந்துரை:

Angular தற்போதைய LTS (நீண்ட கால ஆதரவு) பதிப்புடன் (18 அல்லது 20) சிறப்பாக செயல்படுகிறது.

npm ஐச் சரிபார்க்கவும்

npm -v

npm என்பது Node இன் பேக்கேஜ் மேலாளர் ஆகும்.

இது உங்கள் npm பதிப்பை அச்சிடும், எடுத்துக்காட்டாக:

10.x.x

தானாக நிறுவுதல்:

npm பொதுவாக Node.js உடன் தானாகவே நிறுவப்படும்.

பிழை தீர்வு:

"கட்டளை கண்டுபிடிக்கப்படவில்லை" பிழை கிடைத்தால், Node.js ஐ நிறுவவும் (விவரங்களுக்கு எங்கள் Node.js டுடோரியலைப் பார்க்கவும்).

Angular CLI ஐ நிறுவவும்

Angular CLI என்பது Angular பயன்பாடுகளை உருவாக்க, கட்டமைக்க மற்றும் சேவை செய்வதற்கான அதிகாரப்பூர்வ கருவியாகும்.

CLI (கட்டளை வரி இடைமுகம்)

  • திட்டங்களை உருவாக்குகிறது
  • உருவாக்க சேவையகத்தை இயக்குகிறது
  • உதவிகரமான கட்டளைகளை வழங்குகிறது

குளோபல் நிறுவுதல்

Angular CLI ஐ குளோபலாக நிறுவவும்:

npm install -g @angular/cli

குளோபலாக நிறுவுவது ng கட்டளை எல்லா இடங்களிலும் கிடைக்கச் செய்கிறது.

மாற்று வழி

நீங்கள் குளோபல் நிறுவல் இல்லாமல் npx ஐப் பயன்படுத்தலாம்.

அனுமதி பிழைகள் ஏற்பட்டால், npx ஐப் பயன்படுத்தவும் அல்லது உயர்ந்த அதிகாரங்களுடன் உங்கள் டெர்மினலை இயக்கவும்.

Angular CLI ஐச் சரிபார்க்கவும்

ng உங்கள் PATH இல் உள்ளது மற்றும் பதிப்புகள் பொருந்தக்கூடியவை என்பதை உறுதிப்படுத்த நிறுவலைச் சரிபார்க்கவும்.

ng version

Angular CLI, Node.js மற்றும் Angular பேக்கேஜ் பதிப்புகளைக் காட்டுகிறது:

Angular CLI: 18.x.x
Node: 20.x.x
Package Manager: npm 10.x.x

PATH என்றால் என்ன?

PATH என்பது உங்கள் கணினி ng போன்ற கட்டளைகளுக்காக தேடும் கோப்புறைகளின் பட்டியலாகும்.

ஒரு Angular பயன்பாட்டை உருவாக்கவும்

my-angular-app என்ற பெயரில் ஒரு திட்டத்தை உருவாக்கவும்:

ng new my-angular-app

இது திட்ட கோப்புகள் மற்றும் கோப்புறைகளை உருவாக்கி சார்புகளை நிறுவுகிறது.

நேரம் எடுக்கும்:

இது சில நிமிடங்கள் எடுக்கலாம் மற்றும் சில கேள்விகளைக் கேட்கும் (கீழே பரிந்துரைக்கப்பட்ட தேர்வுகளைப் பார்க்கவும்).

பரிந்துரைக்கப்பட்ட தேர்வுகள்:

  • ஸ்டைல்ஷீட் வடிவம்: CSS (பின்னர் மாற்றலாம்)
  • சேவையக-பக்க ரெண்டரிங் (SSR): இப்போதைக்கு இல்லை (பின்னர் ng add @angular/ssr உடன் சேர்க்கலாம்)
  • Zoneless பயன்பாடு: இப்போதைக்கு இல்லை (பின்னர் ng add @angular/zoneless உடன் மாற்றலாம்)
  • மற்ற கேள்விகள்: இயல்புநிலைகளை வைத்திருங்கள்

மாற்று வழி:

நீங்கள் CLI ஐ குளோபலாக நிறுவ விரும்பவில்லை என்றால், npx ஐப் பயன்படுத்தலாம்:

npx @angular/cli@latest new my-angular-app

Angular பயன்பாட்டை இயக்கவும்

திட்ட கோப்புறைக்குச் செல்லவும்

cd my-angular-app

cd உங்கள் புதிய திட்ட கோப்புறைக்குள் நுழைகிறது.

பயன்பாட்டைத் தொடங்கவும்

ng serve

ng serve ஒரு உள்ளூர் உருவாக்க சேவையகத்தைத் தொடங்குகிறது மற்றும் மாற்றங்களுக்குக் கண்காணிக்கிறது.

முதல் கட்டமைப்பு:

முதல் கட்டமைப்பு ஒரு நிமிடம் எடுக்கலாம்.

✔ Compiled successfully.
✔ Browser application bundle generation complete.

தானாகத் திறக்க (விருப்பம்)

ng serve --open

நீங்கள் உங்கள் இயல்புநிலை உலாவியில் பயன்பாட்டை தானாகத் திறக்க --open கொடியைச் சேர்க்கலாம்.

இயல்புநிலை முகவரி:

இயல்பாக உருவாக்க சேவையகம் http://localhost:4200 இல் இயங்குகிறது.

பயிற்சி

உங்கள் அறிவைச் சோதிக்க இந்தப் பயிற்சியை முயற்சிக்கவும்:

❓உங்கள் கணினியில் Node.js நிறுவப்பட்டுள்ளதா என்று சரிபார்க்க எந்த கட்டளையைப் பயன்படுத்தலாம்?

node -v
✓ சரி! node -v கட்டளை உங்கள் நிறுவப்பட்ட Node.js பதிப்பைக் காட்டுகிறது.
node.js installed
✗ தவறானது! node.js installed என்பது சரியான கட்டளை அல்ல.
npm installed
✗ தவறானது! npm installed என்பது சரியான கட்டளை அல்ல. npm ஐச் சரிபார்க்க npm -v ஐப் பயன்படுத்தவும்.

கட்டளை குறிப்புகள்:

கட்டளை வரி கருவிகளுடன் பணிபுரியும் போது, எப்போதும் கட்டளைகளின் சரியான தொடரியலைச் சரிபார்க்கவும். node -v மற்றும் npm -v கட்டளைகள் பொதுவாக Node.js சரிபார்ப்பிற்குப் பயன்படுத்தப்படுகின்றன.
